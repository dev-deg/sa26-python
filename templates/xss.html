{% if vulnerable %} Why this is vulnerable: The name value is passed to the template with the | safe filter, which tells Jinja2 to trust the string and inject it without encoding. Any <script> tag or event handler becomes live HTML and executes immediately. The cookies on this page are set without HttpOnly — so document.cookie exposes them to the injected script. {% else %} Why this is secure: Jinja2 auto-escapes all variables by default. < becomes &lt;, > becomes &gt;, and so on — the browser renders them as visible text, never as markup. The cookies on this page are set with HttpOnly=True, so even if XSS were somehow possible, document.cookie would return an empty string. {% endif %}
